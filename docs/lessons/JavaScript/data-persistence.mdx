# Module 4.3: Data Persistence 🧠

**Course Code:** ITE-207 | Web System and Technologies 1

## Introduction

Imagine a game that resets your high score every time you refresh the page. Or a Facebook profile that forgets your name every time you log back in. Frustrating, right?

Before this module, our websites had **"Short-term Memory"**—everything was wiped clean when the browser refreshed. Today, we give our websites **"Long-term Memory"** (Data Persistence) and teach them how to talk to other computers on the internet.

---

## 1. JSON: The Universal Language 📦

Computers are like people from different countries. To communicate, they need a common language. In the world of web data, that language is **JSON (JavaScript Object Notation)**.

:::info The "Shipping Crate" Analogy
Think of your data as a physical object (like a chair). You can't just throw a chair through a telephone wire.

1. **Packing:** You disassemble the chair and put it in a box (**`JSON.stringify`**).
2. **Shipping:** You send the box (text) through the wire.
3. **Unpacking:** The receiver opens the box and reassembles the chair (**`JSON.parse`**).
   :::

### Essential Methods

- **`JSON.stringify(object)`**: Converts a JavaScript Object into a **String** (text) so it can be saved or sent.
- **`JSON.parse(string)`**: Converts a **String** back into a JavaScript **Object** so we can use its properties (like `.name` or `.price`).

```javascript
// 1. Our JavaScript Object (The "Chair")
const student = {
  name: "Alden",
  course: "ITE-207",
  isEnrolled: true,
};

// 2. PACKING (Converting to text)
const packedData = JSON.stringify(student);
console.log(packedData);
// Output: '{"name":"Alden","course":"ITE-207","isEnrolled":true}'
// ⚠️ Notice it is now a STRING wrapped in quotes!

// 3. UNPACKING (Converting back to Object)
const unpackedData = JSON.parse(packedData);
console.log(unpackedData.name); // Output: "Alden"
```

## 2. LocalStorage: Your Browser's Backpack 🎒

**LocalStorage** is a small storage space (about 5MB-10MB) inside the user's browser. Data saved here stays there **forever**—even if you close the tab, turn off the computer, or go on vacation!

### Key Features

- **Persistent:** It doesn't expire.
- **Strings only:** You can only store text (This is why JSON is important!).
- **Domain-specific:** Data saved on _your-site.com_ cannot be read by _facebook.com_.

### How to use it

Think of it like a locker with **Labels (Keys)** and **Items (Values)**.

```javascript
// 1. Saving an item (Key, Value)
localStorage.setItem("username", "Derf");
localStorage.setItem("theme", "dark-mode");

// 2. Retrieving an item (By Key)
const user = localStorage.getItem("username");
console.log("Welcome back, " + user); // Output: Welcome back, Derf

// 3. Removing an item
localStorage.removeItem("theme");

// 4. Clearing everything
// localStorage.clear();

```
:::tip 
**How to View LocalStorage Data**
1. Open **Developer Tools** (F12) 
2. **Application** tab  
3. **Local Storage** (on the left sidebar). 

For more details, check the **[Official Chrome DevTools Documentation](https://developer.chrome.com/docs/devtools/storage/localstorage)**.
:::

:::warning 
**Pro Tip: Storing Objects**
Since LocalStorage only saves strings, you **must** stringify objects before saving them!

```javascript
const player = { score: 100 };
localStorage.setItem("saveData", JSON.stringify(player)); // Pack and Save
```

:::

## 3. Fetch API: Talking to the World 📞

The **Fetch API** is how your website asks for information from other servers (APIs). It's like calling a restaurant to ask for the menu.

### Basic Syntax (The "Promise" Way)

```javascript
fetch("https://jsonplaceholder.typicode.com/users/1")
  .then((response) => response.json()) // Wait for the "box" to arrive and unpack it
  .then((data) => {
    console.log("Found user:", data.name);
  })
  .catch((error) => console.log("Something went wrong!", error));
```

---

## 4. Async/Await: The "Wait for it..." Syntax ⏳

While `.then()` works, it can get messy. Modern JavaScript uses `async` and `await` to make our code look like a simple list of instructions.

:::tip 
**The "Coffee Shop" Analogy**

- **`async`**: Tells JS, "This function will take some time, don't freeze the whole app."
- **`await`**: Tells JS, "Pause right here and wait for the coffee to be ready before moving to the next line."
  :::

### Cleaner Code Example

```javascript
async function getStudentData() {
  try {
    // 1. Call the API and WAIT for the response
    const response = await fetch(
      "https://jsonplaceholder.typicode.com/users/1",
    );

    // 2. Unpack the JSON and WAIT for it to finish
    const data = await response.json();

    // 3. Use the data!
    console.log("Student Name:", data.name);
  } catch (err) {
    console.error("Failed to fetch data:", err);
  }
}

getStudentData();
```

### Why use Async/Await?

1.  **Readability:** It reads from top-to-bottom like regular code.
2.  **Error Handling:** We can use `try...catch` blocks just like in regular JS.
3.  **No "Callback Hell":** No more nesting ten `.then()` functions inside each other.
